# Security Policy

This repository (Garudex-Labs / scm) values responsible disclosure and aims to respond promptly to security reports.

Repository: https://github.com/Garudex-Labs/scm  
Organization: https://github.com/Garudex-Labs  
Contact: support@garudexlabs.com

## Scope
This policy applies to the code, services, and infrastructure published in this repository. It does not cover third‑party services or dependencies; report vulnerabilities in them to their maintainers.

## Reporting a Vulnerability (preferred)
1. Use GitHub’s "Report a vulnerability" button under the repository "Security" tab. This creates a private channel between reporter and maintainers.
2. If you cannot use GitHub’s workflow, send details to support@garudexlabs.com.

Please include:
- Affected component and version
- A clear description and proof of concept (steps to reproduce)
- Impact assessment and suggested mitigation
- Your contact information and preferred disclosure timeline

Do not include exploit code or sensitive data in public issues.

## Response and Expectations
- We acknowledge reports within 3 business days.
- We will assess severity, propose mitigations, and keep the reporter informed.
- Fixes are prioritized based on severity.

## Supported Versions & Fixes
- Security and bug fixes are generally provided for the latest minor version only.
- Fixes are released as part of the next minor release or as an on-demand patch release.
- Critical security fixes may prompt an immediate patch release.

## Disclosure
We follow coordinated disclosure. Public disclosure will be agreed upon with the reporter; we may publicly disclose after a fix or after an agreed timeline.

## Bounty / Compensation
This project is currently non‑commercial and run in the public interest. There is no formal bounty program at this time. Contributions of fixes, bug reports, and responsible disclosures are appreciated. If a bounty program is adopted in future, details will be published.

## Contributing fixes
Pull requests that fix security issues are welcome. If the issue is sensitive, coordinate privately (via the GitHub security report workflow or support@garudexlabs.com) before opening a public PR.

Thank you for helping keep this project secure.
